
```bash

```

### **7. Anpassung der Dokumentation**
```bash
# Pfade in Markdown-Dateien ersetzen
find ~/opencloud/coffeeproject/Projekt\ CodeBender/Resources\ Fitness/ -type f -name "*.md" \
  -exec sed -i 's|/home/bastian/ownCloud/Persönlich/coffeeproject/Projekt Fitness/|/home/git/ProjektFitness/|g' {} \;
```

## **Risiken**

| **Risiko**                          | **Lösungsansatz**                                                                 |
|--------------------------------------|-----------------------------------------------------------------------------------|
| Datenverlust bei `rsync`             | Trockenlauf mit `--dry-run` testen. Backup des Quellverzeichnisses erstellen.     |
| Berechtigungsprobleme                | `chmod 755 /home/git/ProjektFitness` und `chown -R bastian:bastian /home/git/ProjektFitness` ausführen. |
| Pfadkonflikte in Dokumentation      | `grep -r "/home/bastian/ownCloud" ~/opencloud/coffeeproject/Projekt\ CodeBender/Resources\ Fitness/` zur Validierung nutzen. |
| Datenbank nicht erreichbar          | Pfad in Konfiguration relativ (`./database/fitness.db`) statt absolut hinterlegen. |

## **Rollback**

1. **Code-Rollback:**
   ```bash
   cd /home/git/ProjektFitness
   git reset --hard
   rm -rf /home/git/ProjektFitness
   ```

2. **Datenbank-Rollback:**
   ```bash
   rm -rf /home/git/ProjektFitness/database
   ```

3. **Dokumentation-Rollback:**
   ```bash
   git -C ~/opencloud/coffeeproject/Projekt\ CodeBender/ checkout -- Resources\ Fitness/
   ```
